4 Nguyên Tắc Cốt Lõi (OpenGitOps)

📜

1. Declarative

Mô tả trạng thái "muốn gì" thay vì ra lệnh "làm gì". Ví dụ: Khai báo replicas: 3 thay vì chạy lệnh kubectl scale.

💾

2. Versioned & Immutable

Mọi thay đổi hạ tầng đều phải được lưu trong Git. Có lịch sử rõ ràng, biết ai sửa khi nào, và dễ dàng rollback bằng git revert.

🧲

3. Pulled Automatically

Agent nằm bên trong Cluster (Argo/Flux) tự động kéo trạng thái từ Git về. An toàn hơn mô hình Push vì CI/CD không cần giữ chìa khóa của Cluster.

🔄

4. Continuously Reconciled

Hệ thống tự động phát hiện sự sai lệch (Drift) giữa Git và Cluster. Nếu ai đó sửa tay trên Cluster, Agent sẽ tự động kéo về đúng trạng thái trên Git.

Công Cụ GitOps & CI/CD

🐙

Argo CD

Công cụ GitOps có giao diện Web UI trực quan, mạnh mẽ và phổ biến nhất.

  • Quản lý trạng thái bằng Application CRD.
  • Mô hình App of Apps để Bootstrap cụm K8s.
  • Tự phục hồi drift với Self Heal.
  • Trạng thái: Synced, OutOfSync, Healthy, Degraded.
⚡

Flux CD

Công cụ GitOps Kubernetes-native, thiết kế theo hướng phân tán với các Controller nhỏ.

  • Gồm các Source, Kustomize, và Helm Controller.
  • Triển khai dễ dàng bằng lệnh flux bootstrap github.
  • Tính năng Image Automation tự cập nhật image mới.
⚙️

GitHub Actions

Robot tự động hóa (CI) thực hiện Build và Test sau mỗi sự kiện Push/PR.

  • Cấu trúc: Workflow → Job → Step.
  • Sử dụng Secrets để bảo mật thông tin.
  • Tạo Docker Image và tự cập nhật mã Git cho Argo/Flux.

Built with ❤️ using GitHub Actions, Argo CD, and Kubernetes.
